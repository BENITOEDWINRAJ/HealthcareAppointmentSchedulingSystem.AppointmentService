
Healthcare Appointment Scheduling System

AppointmentService

AppointmentService is a .NET Core Web API microservice responsible for managing doctor–patient appointments in the healthcare system.

This service works together with PatientService and uses JWT authentication for secure access.

The AppointmentService allows:

Patients to create appointments

Doctors to manage appointments

Doctors to search appointments within a time range

Doctors to update or delete appointments

Fetch registered users from PatientService

API Endpoints
1.Get Appointments

Get appointments for the logged-in user.

Doctor → Gets all appointments

Patient → Gets only their appointments

GET /api/appointments

Authorization:

Bearer Token Required
Roles: Doctor, Patient

Response

{
  "loggedInUserId": "GUID",
  "role": "Doctor",
  "data": [],
  "message": "Appointments retrieved successfully"
}

2.Create Appointment
Create a new appointment.

Both Doctor and Patient can create appointments.

However:

A Patient can only create an appointment for themselves.

POST /api/appointments

Request

{
  "patientId": "GUID",
  "doctorId": "GUID",
  "startTime": "2026-03-14T17:53:49Z",
  "endTime": "2026-03-14T18:53:49Z",
  "status": "Pending"
}

Response

{
  "loggedInUserId": "GUID",
  "username": "doctor@gmail.com",
  "role": "Doctor",
  "data": "APPOINTMENT_ID",
  "message": "Appointment created successfully"
}

3.Search Appointments

Doctors can search appointments by time range with pagination.

GET /api/appointments/search

Query Parameters

doctorId
start
end
page
pageSize

Example

GET /api/appointments/search?doctorId=GUID&start=2026-03-01&end=2026-03-30&page=1&pageSize=5

Response

{
  "doctorId": "GUID",
  "role": "Doctor",
  "data": [],
  "message": "Appointments retrieved successfully for the doctor."
}

4.Update Appointment

Only Doctors can update appointments.

PUT /api/appointments/{id}

Request

{
  "id": "GUID",
  "patientId": "GUID",
  "doctorId": "GUID",
  "startTime": "2026-03-14T17:00:00",
  "endTime": "2026-03-14T18:00:00",
  "status": "Confirmed"
}

Response

{
  "message": "Appointment updated successfully",
  "data": true
}

5.Delete Appointment

Only Doctors can delete appointments.

DELETE /api/appointments/{id}

Response

{
  "message": "Appointment deleted successfully",
  "data": true
}

Get All Registered Users

This endpoint calls PatientService to retrieve registered users.

GET /api/users

Authorization

Role: Doctor

Response

{
  "data": [
    {
      "id": "GUID",
      "username": "doctor@gmail.com",
      "role": "Doctor"
    }
  ],
  "message": "Registered users retrieved successfully"
}

Authentication

AppointmentService uses JWT Authentication.

Tokens are generated by PatientService.

Example header:

Authorization: Bearer <JWT_TOKEN>
Microservice Communication

AppointmentService communicates with PatientService using HttpClient.

The UsersController in AppointmentService calls PatientService to fetch registered users.

Healthcare Appointment Scheduling System

This project implements a Microservices-based Healthcare Appointment System built with ASP.NET Core Web API.It allows patients to schedule appointments with healthcare providers, manage their appointments.

It consists of two independent services:

PatientService – Handles user registration, authentication, and user management.

AppointmentService – Handles appointment scheduling between patients and doctors.

Both services communicate securely using JWT authentication and HTTP-based service-to-service communication.

| Service            | Responsibility                               |
| ------------------ | -------------------------------------------- |
| PatientService     | User registration, login, JWT authentication |
| AppointmentService | Appointment booking and management           |

If you face any challenges while running the solution, please follow the steps below:

Download the source code from the Git repository.

Create a new folder in the root of the C or D drive. Name the folder using a short name like "repo".

Rename the project folder if necessary by removing “-master” from the project name.

Open the solution in Visual Studio. Right-click on the solution and select Open in Terminal.

Run the restore command:

dotnet restore

Clean the solution.

Delete the bin and obj folders in all projects.

Rebuild the solution.

If the build is successful, the solution should run correctly. If the build fails, unload the Test Project and try building the solution again.

If you encounter additional issues, try the following fixes:

Remove the Kafka package from the Infrastructure project if it causes dependency issues.

If the JWT token DLL is not loading properly, remove the JWT reference from the project and add the Shared project reference again. Then rebuild the solution.

I hope these steps help you run the solution successfully.

Note:
Sometimes the Test Project may be blocked by the system. In that case, allow or unblock the repository path because it might be marked as quarantined. After allowing it, try rebuilding the entire solution again.
